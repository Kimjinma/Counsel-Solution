프로젝트 소개

원활한 상담 신청과 상담 기관의 효율적인 관리를 돕기 위해 개발된 웹 기반 종합 상담 솔루션 서비스입니다.

학생들이 언제 어디서든 상담 프로그램을 확인하고 신청할 수 있는 시스템이 필요하다는 기관 측의 실제 요구사항에서 출발했습니다. 수기나 파편화된 시스템으로 관리되던 상담 이력, 프로그램 예약, 그리고 사후 만족도 조사(CS)를 하나의 프로세스로 통합하여 관리자와 학생 모두의 편의성을 높이는 것을 목표로 했습니다.



현재 이 레포지토리는 팀 프로젝트 중 제가 전담했던 핵심 인증(Auth) 및 마이페이지/CS 후속 관리 기능을 중심으로 발췌하여 코드를 다듬은 버전입니다.



# 기술 스택



- **Backend**: Java 17, Spring Boot, Spring Data JPA, Spring Security (OAuth2)

- **Database**: MySQL

- **Template Engine**: Mustache, Thymeleaf

# 주요 기능 및 고민한 점



### 1. OAuth2 기반 소셜 통합 인증 및 사용자 보안 처리

- 무분별한 계정 생성을 막고 학생들의 접근성을 높이기 위해 Spring Security와 OAuth2(Google, Naver)를 활용해 소셜 로그인과 일반 로그인을 통합적으로 구현했습니다.

- 비밀번호 분실 시 이메일 전송을 통한 임시 비밀번호 발급 로직을 구현했으며, 사용자 정보를 안전하게 보호하기 위해 BCrypt로 비밀번호를 단방향 해시 암호화하여 DB에 저장하도록 설계했습니다. 

- 동시 가입 시 식별값 충돌을 막기 위해 가입 ID는 UUID 기반으로 안전하게 발급하도록 적용했습니다.



### 2. 프로그램 예약 및 마이페이지 이력 관리 연동

- 학생이 상담 프로그램을 신청하면 본인의 마이페이지에서 예약 상태(승인/대기)를 확인할 수 있는 체계를 만들었습니다.

- 학생 번호와 프로그램 번호 등 여러 개의 키를 조합해야 하는 외래키 의존성을 해결하기 위해 JPA 엔티티 간 연관 관계 매핑(ManyToOne)을 적용하여, 학생 정보 조회 시 상담 이력이 자연스럽게 따라오도록 객체 지향적으로 구성했습니다.



### 3. 상담 사후 만족도 조사(CS) 데이터 무결성 보장

- 상담이 만료 혹은 완료된 건에 대해서만 만족도(객관식 점수 및 주관식 리뷰) 조사를 등록할 수 있도록 예외 처리와 검증 로직을 추가했습니다.

- 다중 질문에 대한 주관식 데이터가 DB에 저장될 때, 필드 매핑 오류로 인해 마지막 값만 덮어씌워지는 데이터 유실 문제를 발견하였습니다. 각 주관식 항목을 개별 컬럼으로 완벽히 분리 저장하도록 서비스 로직과 모델(Entity) 구조를 리팩토링해 데이터 무결성을 확보했습니다.

# 코드 품질 개선 및 한계점 극복 노력



1년 전 개발 당시에는 병합 과정에서 발생한 Git Conflict 마커를 방치하거나, 디버깅을 위한 `System.out.println`을 전역에 흩뿌리고, DB 접속 비밀번호를 `application.properties` 파일에 하드코딩하는 등 기본기가 부족했습니다.



이를 부끄러운 과거로 숨기지 않고 취업 전 **오답 노트**로 삼아 코드를 리팩토링했습니다. 

- 민감 정보는 시스템 환경 변수로 격리 조치했습니다.

- 컴파일 오류가 발생해 돌아가지 않거나 누락되어 있던 레거시 버그들을 걷어내고 안정화했습니다.
